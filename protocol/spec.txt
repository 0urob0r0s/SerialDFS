SerialDFS Protocol Specification — v1
======================================
Locked: 2026-04-25  (ADR-0003)

1. Serial Frame Layout
----------------------
All integers little-endian.

  Offset  Size  Field        Type       Notes
  ------  ----  -----------  ---------  ----------------------------------------
    0       2   magic        char[2]    Always 'S' (0x53) 'D' (0x44)
    2       1   version      uint8      v1 = 1; reject frames where != 1
    3       1   flags        uint8      Bit flags (see §4)
    4       1   seq          uint8      Wrapping request/response sequence ID
    5       1   command      uint8      Command ID (see §2); set in both req/resp
    6       1   status       uint8      Response status; 0x00 in requests (see §3)
    7       1   reserved     uint8      Must be 0x00 in v1
    8       2   payload_len  uint16le   Payload byte count, 0..512
   10       N   payload      bytes      Command-specific (see §5)
  10+N      2   crc16        uint16le   CRC-16/CCITT-FALSE over bytes [0..10+N-1]

Total frame size: 12 + N bytes  (N = payload_len)
Maximum payload: 512 bytes
Minimum frame:    12 bytes (N=0)


2. CRC Variant (locked, ADR-0003)
----------------------------------
  Algorithm  : CRC-16/CCITT-FALSE
  Polynomial : 0x1021
  Init value : 0xFFFF
  Reflect in : No
  Reflect out: No
  XOR out    : 0x0000
  Check value: 0x29B1  (CRC of ASCII "123456789")

Reference: protocol/crc16_reference.py
Both dos/src/serframe.c and linux/serdfsd/frame.py use a 256-entry
table built from the same polynomial and init value.


3. Command IDs
--------------
  Name             ID    Direction    Purpose
  ---------------  ----  -----------  -----------------------------------
  PING             0x01  DOS->Linux   Connectivity and protocol sanity
  GET_INFO         0x02  DOS->Linux   Server capabilities/limits
  LIST_DIR_BEGIN   0x10  DOS->Linux   Start directory enumeration
  LIST_DIR_NEXT    0x11  DOS->Linux   Continue directory enumeration
  OPEN             0x20  DOS->Linux   Open existing file (read)
  CREATE           0x21  DOS->Linux   Create or truncate file
  READ             0x22  DOS->Linux   Read bytes from open handle
  WRITE            0x23  DOS->Linux   Write bytes to open handle
  SEEK             0x24  DOS->Linux   Move file pointer
  CLOSE            0x25  DOS->Linux   Close open handle
  DELETE           0x30  DOS->Linux   Delete file
  RENAME           0x31  DOS->Linux   Rename file/directory
  MKDIR            0x32  DOS->Linux   Create directory
  RMDIR            0x33  DOS->Linux   Remove empty directory
  GET_ATTR         0x40  DOS->Linux   Read DOS-compatible attributes
  SET_ATTR         0x41  DOS->Linux   Set DOS-compatible attributes
  GET_TIME         0x42  DOS->Linux   Read DOS date/time
  SET_TIME         0x43  DOS->Linux   Set DOS date/time
  FLUSH            0x50  DOS->Linux   Flush buffered writes
  STATUS           0x60  DOS->Linux   Read daemon stats


4. Status IDs
-------------
  Name             ID    DOS error mapping
  ---------------  ----  -----------------
  OK               0x00  Success
  ERR_NOT_FOUND    0x01  File not found (0x02)
  ERR_ACCESS       0x02  Access denied (0x05)
  ERR_BAD_HANDLE   0x03  Invalid handle (0x06)
  ERR_EXISTS       0x04  File already exists (0x50)
  ERR_IO           0x05  General I/O error (0x1D)
  ERR_CRC          0x06  Data error / retryable transport error (0x17)
  ERR_TIMEOUT      0x07  Drive/network not ready (0x15)
  ERR_UNSUPPORTED  0x08  Unsupported function (0x01)
  ERR_BAD_PATH     0x09  Path not found (0x03)
  ERR_PROTOCOL     0x0A  Protocol error — fail closed (0x0B)


5. Flag Bits
------------
  Bit  Mask  Name       Meaning
  ---  ----  ---------  ------------------------------------------
    0  0x01  FLAG_RETRY This packet is a retransmit of the request
  1-7        (reserved) Must be zero in v1


6. Conformance Vectors
----------------------
Golden test vectors: protocol/vectors/
  PNGREQ.BIN   — valid PING request  (seq=1)         -> FRAME_OK
  PNGRESP.BIN  — valid PING response (seq=1, OK)     -> FRAME_OK
  BADCRC.BIN   — PING frame with CRC XOR'd 0xFFFF    -> FRAME_BADCRC
  TRUNC.BIN    — first 5 bytes only                  -> FRAME_SHORT
  STALE.BIN    — valid PING frame (seq=2)             -> FRAME_OK, seq != expected

Each .bin file has a paired .hex.txt annotation.
Validated by: dos/tools/CRCFUZZ.EXE and linux/serdfsd/tests/test_frame.py
